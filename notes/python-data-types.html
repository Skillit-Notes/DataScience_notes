Chapter 2: Python Strings - Working with Text

üë¶
Alex
I need to work with text in my programs. How do strings work in Python?
üëß
Sam
Strings are sequences of characters - think of them as text containers! You can create them with single quotes, double quotes, or triple quotes for multi-line text: 
# Different ways to create strings
name = 'Alex'              # Single quotes
message = "Hello, World!"  # Double quotes
paragraph = """This is a
multi-line string that
spans several lines!"""     # Triple quotes

# Strings are immutable (can't change individual characters)
text = "Python"
# text[0] = 'J'  # This would cause an error!

# But you can create new strings from old ones
new_text = 'J' + text[1:]  # "Jython"
print(new_text)
üë¶
Alex
What does "immutable" mean? And how can I access specific characters?
üëß
Sam
"Immutable" means you can't change a string once it's created - you can only create new strings. Think of it like a printed book - you can't change the letters on the page, but you can write a new book!

For accessing characters, Python uses indexing and slicing: 
word = "Python"

# Indexing (accessing single characters)
print(word[0])    # 'P' (first character)
print(word[2])    # 't' (third character)
print(word[-1])   # 'n' (last character)
print(word[-2])   # 'o' (second to last)

# Slicing (getting substrings)
print(word[0:3])  # 'Pyt' (characters 0,1,2)
print(word[2:])   # 'thon' (from index 2 to end)
print(word[:4])   # 'Pyth' (from start to index 3)
print(word[::2])  # 'Pto' (every 2nd character)
print(word[::-1]) # 'nohtyP' (reversed!)
üë¶
Alex
Cool! What about modifying strings? You said they're immutable, but I need to change text sometimes.
üëß
Sam
Right! While strings are immutable, Python has tons of methods that create NEW strings with the changes you want. These methods don't modify the original - they return a new string: 
text = "  Hello Python!  "

# Case transformations (create new strings)
print(text.upper())       # "  HELLO PYTHON!  "
print(text.lower())       # "  hello python!  "
print(text.title())       # "  Hello Python!  "
print(text.capitalize())  # "  hello python!  "
print(text.swapcase())    # "  hELLO pYTHON!  "

# Trimming whitespace
print(text.strip())       # "Hello Python!" (removes both sides)
print(text.lstrip())      # "Hello Python!  " (left only)
print(text.rstrip())      # "  Hello Python!" (right only)

# Searching and checking
msg = "I love Python programming"
print(msg.find("Python"))      # 7 (index where it starts)
print(msg.count("o"))          # 2 (number of 'o's)
print(msg.startswith("I"))     # True
print(msg.endswith("ing"))     # True

# Replacing and splitting
print(msg.replace("Python", "Java"))  # "I love Java programming"
words = msg.split()                       # ['I', 'love', 'Python', 'programming']
print(words)
print("-".join(words))             # "I-love-Python-programming"
üë¶
Alex

üëß

Python has built-in checking methods that return True or False - perfect for validation! 
# Content checking methods

# Practical example: validating user input
username = input("Enter username (letters/numbers only): ")

    print("Valid username!")
else:
    print("Username can only contain letters and numbers!")

üî§ Essential String Methods

  • Transform: upper(), lower(), title(), capitalize()
  • Clean: strip(), lstrip(), rstrip()
  • Search: find(), index(), count(), in
  • Check: isalpha(), isdigit(), isspace()
  • Modify: replace(), split(), join()

‚ú® String Formatting

  • f-strings: f"Hello {name}" (Python 3.6+)
  • format(): "{} {}".format(a, b)
  • %: "%s %d" % (text, num) (old style)
  • Concatenation: str1 + str2
Memory Tip! Remember: strings are like printed pages - you can read them, but to make changes, you need to print a new page. All string methods return NEW strings!

# Creating lists
shopping = ["apples", "bread", "milk"]
numbers = [10, 20, 30, 40]
mixed = ["Alex", 25, 3.14, True]  # Can mix types!

Alex
How do I access items in a list and add new ones?
üëß
Sam
Lists use the same indexing as strings, but since they're mutable, you can also change items directly: 
fruits = ["apple", "banana", "orange"]

# Accessing items (same as strings)
print(fruits[0])     # "apple" (first item)
print(fruits[-1])    # "orange" (last item)
print(fruits[1:3])   # ["banana", "orange"] (slicing)

# Modifying items (this is new!)
fruits[1] = "grape"    # Change banana to grape
print(fruits)         # ["apple", "grape", "orange"]

# Adding items - three ways!
fruits.append("mango")        # Add one item to end
fruits.extend(["kiwi", "pear"]) # Add multiple items
fruits.insert(1, "cherry")    # Insert at specific position

print(fruits)
üë¶
Alex
What about removing items? And what if I want to find or count things?
üëß
Sam
Lists have several ways to remove items, plus helpful methods for finding and counting: 
colors = ["red", "blue", "green", "blue", "yellow"]

# Removing items - multiple ways!
colors.remove("green")     # Remove first matching value
last = colors.pop()        # Remove & return last item
print(f"Removed: {last}")
item = colors.pop(1)       # Remove & return item at index 1

# Finding and counting

print(numbers.count(20))   # 2 (appears twice)
print(numbers.index(30))   # 2 (position of 30)

print(20 in numbers)      # True (is in list)

# Useful aggregates

üë¶
Alex
Can I sort lists? And what's this "list comprehension" I keep hearing about?

Yes! Lists can be sorted and reversed. And list comprehensions are Python's super-compact way to create lists: 
# Sorting and reversing
nums = [3, 1, 4, 1, 5, 9]
nums.sort()                # Sort in place (modifies list)
print(nums)               # [1, 1, 3, 4, 5, 9]

# List comprehensions - create lists in one line!

# Transform existing list
words = ["hello", "world", "python"]
upper_words = [w.upper() for w in words]
print(upper_words)  # ['HELLO', 'WORLD', 'PYTHON']

  • Remove: remove(), pop(), del, clear()
  • Find: index(), count(), in
  • Order: sort(), reverse()
  • Copy: copy() or list[:]

üí° List vs String

  • Both: Ordered, indexable, sliceable
  • Lists: Mutable, any types, [ ]
  • Strings: Immutable, only characters, " "
  • Lists can contain lists (nested)
Best Practice! Use descriptive names for lists (plural nouns work well): students not s, prices not p. This makes your code self-documenting!

Chapter 4: Python Tuples - Immutable Sequences

üë¶
Alex
Sam, I've heard about tuples. How are they different from lists?
üëß
Sam
Tuples are like lists' responsible older sibling - they're ordered collections, but they're immutable (can't be changed). Think of them as "protected lists" for data that shouldn't change: 
# Creating tuples - use parentheses
point = (3, 4)              # 2D coordinates
rgb = (255, 128, 0)        # Color values
person = ("Alice", 25, "Engineer")  # Mixed types

# Single item tuple needs a comma!
single = (42,)    # This is a tuple
not_tuple = (42)  # This is just the number 42!

# Empty tuple
empty = ()

# Tuples can be created without parentheses
coordinates = 5, 10  # Still a tuple!
print(type(coordinates))  # <class 'tuple'>
üë¶
Alex
So I can't change them? Why would I want that?
üëß
Sam
Immutability is actually a superpower! It makes your data safer and your code more predictable. Here's why tuples are useful:

1. Data integrity - Some data shouldn't change (like coordinates, database records)
2. Dictionary keys - Tuples can be dictionary keys (lists can't!)
3. Performance - Tuples are faster than lists
4. Multiple returns - Functions can return multiple values as tuples

Let me show you: 
# Accessing tuple items (same as lists)
point = (10, 20)
x = point[0]  # 10
y = point[1]  # 20

# Trying to change raises an error
# point[0] = 15  # TypeError: tuple doesn't support item assignment

# Tuple unpacking - super useful!
x, y = point  # Unpack into separate variables
print(f"x={x}, y={y}")

# Multiple assignment in one line
name, age, job = ("Bob", 30, "Teacher")

# Swapping variables elegantly
a, b = 5, 10
a, b = b, a  # Swap values!
print(f"a={a}, b={b}")  # a=10, b=5
üë¶
Alex
That unpacking looks really useful! Can you show me more advanced examples?
üëß
Sam
Absolutely! Tuple unpacking is one of Python's most elegant features. Here are some powerful patterns: 
# Extended unpacking with *
numbers = (1, 2, 3, 4, 5)
first, *middle, last = numbers
print(f"first={first}, middle={middle}, last={last}")
# first=1, middle=[2, 3, 4], last=5

# Returning multiple values from functions
def get_min_max(numbers):
    return min(numbers), max(numbers)  # Returns a tuple

low, high = get_min_max([3, 7, 2, 9, 1])
print(f"Range: {low} to {high}")

# Tuples as dictionary keys (lists can't do this!)
locations = {}
locations[(40.7, -74.0)] = "New York"
locations[(51.5, -0.1)] = "London"
print(locations[(40.7, -74.0)])  # "New York"

# Named tuples (more readable!)
from collections import namedtuple
Person = namedtuple('Person', ['name', 'age', 'job'])
alice = Person("Alice", 30, "Developer")
print(alice.name)  # More readable than alice[0]
üë¶
Alex
Wait, you said tuples can contain mutable objects. Doesn't that break immutability?
üëß
Sam
Excellent observation! The tuple itself is immutable (you can't add/remove items or reassign positions), but if it contains mutable objects, those objects can still be modified: 
# Tuple with mutable object (list)
data = ("fixed", [1, 2, 3], 42)
# data[0] = "changed"  # Error! Can't reassign
# But we CAN modify the list inside:
data[1].append(4)
print(data)  # ("fixed", [1, 2, 3, 4], 42)

# Tuple methods (only 2!)
numbers = (1, 3, 2, 3, 3, 4)
print(numbers.count(3))  # 3 (appears 3 times)
print(numbers.index(2))  # 2 (position of first 2)

üì¶ When to Use Tuples

  • Fixed collections (coordinates, RGB values)
  • Function returns with multiple values
  • Dictionary keys (immutable required)
  • Data that shouldn't change
  • Performance-critical code

üîÑ Tuple vs List

  • Tuple: Immutable, (), faster, hashable
  • List: Mutable, [], more methods, flexible
  • Use tuples for structure, lists for collections
Remember! If you need to change data ‚Üí use a list. If data should stay constant ‚Üí use a tuple. When in doubt, start with a list and refactor to tuple if needed.

Chapter 5: Python Sets - Unique Collections

üë¶
Alex
Sam, I have a list with duplicates and I want only unique items. Is there an easy way?
üëß
Sam
Perfect use case for sets! Sets are collections that automatically keep only unique items - no duplicates allowed. They're like mathematical sets you learned in school: 
# Creating sets - duplicates are automatically removed!
numbers = {1, 2, 3, 2, 1, 4}
print(numbers)  # {1, 2, 3, 4} - duplicates gone!

# Convert list to set to remove duplicates
scores = [85, 90, 85, 75, 90, 95]
unique_scores = set(scores)
print(unique_scores)  # {75, 85, 90, 95}

# Empty set (careful with syntax!)
empty_set = set()  # Can't use {} - that's an empty dict!

# Set from string (unique characters)
letters = set("hello")
print(letters)  # {'h', 'e', 'l', 'o'}
üë¶
Alex
Cool! But wait, the order changed. And can I access items by index like lists?
üëß
Sam
Good catch! Sets are unordered - they don't maintain any specific order, and you can't access items by index. But they have their own superpowers:

1. Lightning-fast membership testing (checking if item exists)
2. Set operations (union, intersection, difference)
3. Automatic duplicate removal

Let me show you set operations - they're really powerful: 
# Adding and removing items
fruits = {"apple", "banana"}
fruits.add("orange")         # Add one item
fruits.update(["grape", "kiwi"])  # Add multiple
print(fruits)

fruits.discard("banana")     # Remove (no error if missing)
fruits.remove("grape")       # Remove (error if missing)
popped = fruits.pop()        # Remove random item

# Fast membership testing
print("apple" in fruits)   # Super fast!

# Set mathematics!
set_a = {1, 2, 3, 4}
set_b = {3, 4, 5, 6}

# Union (all unique items from both)
print(set_a | set_b)        # {1, 2, 3, 4, 5, 6}
print(set_a.union(set_b))   # Same result

# Intersection (common items)
print(set_a & set_b)        # {3, 4}
print(set_a.intersection(set_b))

# Difference (in a but not in b)
print(set_a - set_b)        # {1, 2}
print(set_a.difference(set_b))

# Symmetric difference (in either, but not both)
print(set_a ^ set_b)        # {1, 2, 5, 6}
üë¶
Alex
Wow, those set operations are like Venn diagrams! When would I use these in real code?
üëß
Sam
Exactly like Venn diagrams! Here are practical examples: 
# Example 1: Find common friends
alice_friends = {"Bob", "Charlie", "David", "Eve"}
bob_friends = {"Alice", "Charlie", "Eve", "Frank"}

common = alice_friends & bob_friends
print(f"Common friends: {common}")  # Charlie, Eve

# Example 2: Find unique words in text
text = "the quick brown fox jumps over the lazy dog"
unique_words = set(text.split())
print(f"Unique words: {len(unique_words)}")

# Example 3: Check if sets are related
required_skills = {"Python", "SQL", "Git"}
candidate_skills = {"Python", "SQL", "Git", "Docker"}

if required_skills.issubset(candidate_skills):
    print("Candidate has all required skills!")

# Set comprehension
squares = {n**2 for n in range(10)}
print(squares)  # {0, 1, 4, 9, 16, 25, 36, 49, 64, 81}

üéØ Set Use Cases

  • Remove duplicates from data
  • Fast membership testing
  • Find common/different elements
  • Track unique items
  • Mathematical set operations

‚ö° Set Performance

  • in operator: O(1) average
  • Add/remove: O(1) average
  • Much faster than lists for membership
  • frozenset: immutable version
Pro Tip! Use sets when you need uniqueness or fast lookups. Use lists when order matters. Use tuples when data shouldn't change. Each has its perfect use case!

üë¶
Alex
Sam, I need to store information about students - their names, ages, and grades. Lists seem clunky for this.
üëß
Sam
Perfect timing to learn dictionaries! They're like real dictionaries - you look up a key (word) to get a value (definition). They're ideal for structured data: 
# Creating dictionaries
student = {
    "name": "Alex",
    "age": 18,
    "grade": "A",
    "subjects": ["Math", "Physics", "CS"]
}

# Access values by key
print(student["name"])      # "Alex"
print(student["subjects"])  # ["Math", "Physics", "CS"]

# Safer access with get()
print(student.get("email"))  # None (no error)
print(student.get("email", "N/A"))  # "N/A" (default)

# Other ways to create dicts
empty = {}
from_pairs = dict([("a", 1), ("b", 2)])
from_kwargs = dict(name="Bob", age=25)
üë¶
Alex
This is exactly what I need! How do I add, update, and remove items?
üëß
Sam
Dictionaries are mutable, so you can modify them easily: 
person = {"name": "Alice", "age": 25}

# Add new key-value pairs
person["city"] = "Boston"
person["job"] = "Engineer"

# Update existing values
person["age"] = 26

# Update multiple at once
person.update({"age": 27, "city": "Seattle"})

# Remove items
job = person.pop("job")        # Remove and return value
del person["city"]             # Remove key-value pair
last = person.popitem()         # Remove last inserted item

# Check if key exists
if "name" in person:
    print(f"Name is {person['name']}")

# Get all keys, values, or items
data = {"x": 10, "y": 20, "z": 30}
print(list(data.keys()))    # ["x", "y", "z"]
print(list(data.values()))  # [10, 20, 30]
print(list(data.items()))   # [("x", 10), ("y", 20), ("z", 30)]
üë¶
Alex
How do I loop through dictionaries? And can I have nested dictionaries?
üëß
Sam
Yes to both! Looping is elegant, and nesting lets you create complex data structures: 
# Looping through dictionaries
scores = {"Alice": 95, "Bob": 87, "Charlie": 92}

# Loop through keys
for name in scores:
    print(f"{name}: {scores[name]}")

# Loop through key-value pairs (best way!)
for name, score in scores.items():
    print(f"{name} scored {score}")

# Loop through values only
total = sum(scores.values())
print(f"Total: {total}")

# Nested dictionaries (like JSON!)
school = {
    "students": {
        "alice": {"age": 18, "grade": "A"},
        "bob": {"age": 17, "grade": "B"}
    },
    "teachers": {
        "math": "Dr. Smith",
        "physics": "Dr. Jones"
    }
}

# Access nested values
print(school["students"]["alice"]["grade"])  # "A"

# Dictionary comprehension
squares = {n: n**2 for n in range(5)}
print(squares)  # {0: 0, 1: 1, 2: 4, 3: 9, 4: 16}
üë¶
Alex
What about that setdefault() method? And can any value be a key?
üëß
Sam
Great questions! setdefault() is super useful, and keys must be immutable (hashable): 
# setdefault - get value or set default

config.setdefault("theme", "dark")  # Sets and returns "dark"

count = {}
for word in words:
    count[word] = count.get(word, 0) + 1
print(count)  # {"apple": 2, "banana": 1, "cherry": 1}

# Valid dictionary keys (must be immutable)
valid_dict = {
    "string_key": "value",
    42: "integer key",
    3.14: "float key",
    (1, 2): "tuple key",  # Tuples can be keys!
    True: "boolean key"
}

# Invalid keys (mutable types)
# bad_dict = {[1, 2]: "list key"}  # TypeError!
# bad_dict = {{1, 2}: "set key"}   # TypeError!

üìö Dictionary Methods

  • Access: [], get(), setdefault()
  • Modify: update(), pop(), popitem()
  • View: keys(), values(), items()
  • Check: in, len()
  • Copy: copy(), dict()

üéØ Perfect For

  • JSON-like data structures
  • Configurations and settings
  • Caching and memoization
  • Counting and grouping
  • Fast lookups by key
Best Practice! Use meaningful key names. user["email"] is much clearer than user["e"]. Think of dictionaries as mini-databases for your data!

Chapter 7: Type Conversion - Transforming Data

üë¶
Alex
Sam, I keep getting errors when I try to combine different data types. How do I convert between them?
üëß
Sam
Type conversion (or "casting") is essential in Python! You can convert between types using built-in functions. Let me show you the common conversions: 
# String to number conversions
text_num = "42"
integer = int(text_num)      # 42
print(type(integer))        # <class 'int'>

text_float = "3.14"
decimal = float(text_float)  # 3.14

# Be careful with int() on floats in strings!
# int("3.14")  # ValueError! Can't convert directly
# Do this instead:
result = int(float("3.14"))  # 3 (truncates decimal)

# Number to string
num = 123
text = str(num)            # "123"
print(f"The number is {text}")

# Boolean conversions
print(bool(1))          # True (any non-zero)

When I get input from users, it's always a string. How do I safely convert it?

Excellent point! User input needs careful handling. Always validate and use try-except for safety: 
# Safe conversion with error handling
user_input = input("Enter your age: ")

try:
    age = int(user_input)
    print(f"Next year you'll be {age + 1}")
except ValueError:
    print("Please enter a valid number!")

# Function for safe conversion
def safe_int(text, default=0):
    try:
        return int(text)
    except (ValueError, TypeError):
        return default

# Using the safe function
value = safe_int("abc", default=-1)
print(value)  # -1

# Checking before converting
text = "123"
if text.isdigit():
    num = int(text)
    print(f"Converted: {num}")
else:
    print("Not a valid number")

  • To string: str(anything)
  • To int: int(string/float)
  • To float: float(string/int)
  • To list: list(iterable)
  • To tuple: tuple(iterable)
  • To set: set(iterable)
  • To dict: dict(pairs)
  • To bool: bool(value)

  • int("3.14") fails - use int(float("3.14"))
  • set() removes duplicates and order

  • str() on list gives "[1, 2, 3]" not "123"
  • Always handle user input errors
Golden Rule! When converting user input or external data, always use try-except or validate first. It's better to handle errors gracefully than crash your program!
