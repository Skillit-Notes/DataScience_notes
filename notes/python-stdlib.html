📚 Python Standard Library Essentials

Discover the powerful tools that come built into Python!

🗂️ os – Operating System Services

👦
Alex (Exploring the Standard Library)
Sam, I need to work with files and folders in my programs. What's the best way?
👧
Sam (Python Expert)
The os module is your bridge to the operating system! It's like having a remote control for your computer's file system. You can navigate folders, create files, and manage paths safely across different operating systems!
👦
Alex
Show me what it can do!
👧
Sam
Let's explore the file system together: 
import os

# Where are we right now?
print("Current directory:", os.getcwd())

# List files and folders
print("\nDirectory contents:")
for item in os.listdir():
    print(" -", item)

# Create a directory (safely)
folder = "my_project"
if not os.path.exists(folder):
    os.mkdir(folder)
    print(f"\nCreated folder: {folder}")

# Build paths the smart way (works on Windows/Mac/Linux!)
file_path = os.path.join(folder, "data.txt")
print("Safe path:", file_path)

# Write to a file
with open(file_path, "w") as f:
    f.write("Hello from Python!")

# Get file info
size = os.path.getsize(file_path)
print(f"File size: {size} bytes")

# Clean up
os.remove(file_path)
os.rmdir(folder)
print("Cleaned up!")

📁 Common os Operations

  • os.getcwd() - Current directory
  • os.listdir() - List contents
  • os.mkdir() - Create folder
  • os.remove() - Delete file
  • os.path.join() - Build paths

🎯 Why Use os?

  • Cross-platform compatibility
  • Safe path handling
  • File system navigation
  • Environment variables
  • Process management
Pro Tip! Always use os.path.join() for building file paths. It automatically uses the correct separator (/ or \) for your operating system!

⚙️ sys – System-Specific Parameters

👦
Alex
What's the difference between os and sys?
👧
Sam
While os deals with your operating system and files, sys is about the Python interpreter itself! It's like looking under Python's hood - you can see version info, command-line arguments, and how Python is configured on your system.
👦
Alex
When would I need that information?
👧
Sam
Let me show you practical uses: 
import sys

# Python version info
print("Python version:", sys.version)
print("Version info:", sys.version_info)
print("Platform:", sys.platform)

# Command-line arguments
# When you run: python script.py arg1 arg2
print("\nCommand line args:", sys.argv)

# Python's module search path
print("\nPython looks for modules in:")
for path in sys.path[:3]:  # First 3 paths
    print(" -", path)

# Exit with a message (useful in scripts)
def check_python_version():
    if sys.version_info < (3, 6):
        sys.exit("Error: Python 3.6+ required!")
    print("Python version OK!")

check_python_version()

# Memory size of objects
my_list = [1, 2, 3, 4, 5]
print(f"\nList size: {sys.getsizeof(my_list)} bytes")
Script Tip! sys.argv is super useful for command-line tools. sys.argv[0] is always your script name, and sys.argv[1:] are the arguments passed to it!

➗ math – Mathematical Functions

👦
Alex
Python can already do math with +, -, *, /. Why do I need a math module?
👧
Sam
The math module is like upgrading from a basic calculator to a scientific one! You get trigonometry, logarithms, constants like π, and advanced functions. Perfect for geometry, physics, statistics, and any serious mathematical work!
👦
Alex
Show me some cool math tricks!
👧
Sam
Let's explore mathematical wonders: 
import math

# Mathematical constants
print("π =", math.pi)
print("e =", math.e)
print("τ (tau) =", math.tau)  # 2π

# Roots and powers
print("\nsqrt(16) =", math.sqrt(16))
print("2^10 =", math.pow(2, 10))
print("hypot(3,4) =", math.hypot(3, 4))  # √(3²+4²)

# Rounding functions
print("\nfloor(3.7) =", math.floor(3.7))  # Round down
print("ceil(3.1) =", math.ceil(3.1))    # Round up
print("trunc(3.9) =", math.trunc(3.9))  # Remove decimals

# Trigonometry (radians)
angle = math.pi / 6  # 30 degrees
print("\nsin(30°) =", math.sin(angle))
print("cos(30°) =", math.cos(angle))

# Convert degrees ↔ radians
print("180° in radians:", math.radians(180))
print("π radians in degrees:", math.degrees(math.pi))

# Factorial and combinations
print("\n5! =", math.factorial(5))
print("C(5,2) =", math.comb(5, 2))  # Choose 2 from 5
print("P(5,2) =", math.perm(5, 2))  # Permutations

# Logarithms
print("\nlog(e) =", math.log(math.e))  # Natural log
print("log10(100) =", math.log10(100))
print("log2(8) =", math.log2(8))

📐 Essential Functions

  • sqrt(), pow() - Roots & powers
  • sin(), cos(), tan() - Trigonometry
  • floor(), ceil() - Rounding
  • factorial() - n!
  • log(), log10() - Logarithms

🎯 Real-World Uses

  • Game physics calculations
  • Data science & statistics
  • Engineering simulations
  • Graphics & animations
  • Financial calculations
Remember! Trigonometric functions use radians, not degrees. Use math.radians() and math.degrees() to convert between them!

🎲 random – Random Number Generation

👦
Alex
I want to add some unpredictability to my programs. How do I generate random numbers?
👧
Sam
The random module is your dice roller, card shuffler, and lottery picker all in one! It generates pseudo-random numbers - they're not truly random but perfect for games, simulations, and testing. Think of it as controlled chaos!
👦
Alex
Can I make the randomness predictable for testing?
👧
Sam
Yes! That's the beauty of pseudo-random numbers. Let me show you: 
import random

# Set seed for reproducible results
random.seed(42)  # Same seed = same "random" numbers!

# Random float between 0 and 1
print("random():", random.random())

# Random float in a range
print("uniform(1, 10):", random.uniform(1, 10))

# Random integer (inclusive on both ends!)
dice_roll = random.randint(1, 6)
print(f"Dice roll: {dice_roll}")

# Random choice from a list
colors = ["red", "green", "blue", "yellow"]
print("Random color:", random.choice(colors))

# Multiple random choices (no repeats)
lottery = random.sample(range(1, 50), k=6)
print("Lottery numbers:", sorted(lottery))

# Shuffle a list in-place
deck = list(range(1, 11))
random.shuffle(deck)
print("Shuffled deck:", deck)

# Weighted choices (some more likely)
weather = ["sunny", "cloudy", "rainy"]
weights = [60, 30, 10]  # 60% sunny, 30% cloudy, 10% rainy
today = random.choices(weather, weights=weights, k=1)[0]
print(f"Today's weather: {today}")

# Random password generator
import string
chars = string.ascii_letters + string.digits + "!@#$%"
password = ''.join(random.choices(chars, k=12))
print(f"Random password: {password}")

🎯 Common Functions

  • random() - Float 0.0 to 1.0
  • randint(a,b) - Integer a to b
  • choice(seq) - Pick one item
  • shuffle(list) - Mix in-place
  • sample(seq,k) - Pick k unique

🎮 Fun Uses

  • Game mechanics (dice, cards)
  • Random test data
  • Simulations & modeling
  • Password generation
  • Random sampling
Testing Tip! Always use random.seed() when you need reproducible results. This is crucial for debugging and testing!

🌐 json – Working with JSON Data

👦
Alex
I keep hearing about JSON everywhere. What is it exactly?
👧
Sam
JSON (JavaScript Object Notation) is the universal language of data exchange! It's how web APIs talk, how config files are stored, and how different programs share data. Python's json module is your translator between Python objects and JSON text!
👦
Alex
So it converts between Python and text?
👧
Sam
Exactly! Watch how seamlessly it works: 
import json

# Python data structure
data = {
    "name": "Alex",
    "age": 25,
    "skills": ["Python", "SQL", "JSON"],
    "active": True,
    "score": None,
    "address": {
        "city": "New York",
        "zip": "10001"
    }
}

# Convert to JSON string
json_string = json.dumps(data)
print("Compact JSON:")
print(json_string)

# Pretty formatting for humans
pretty_json = json.dumps(data, indent=2, sort_keys=True)
print("\nPretty JSON:")
print(pretty_json)

# Save to file
with open("user_data.json", "w") as f:
    json.dump(data, f, indent=2)
print("\nSaved to user_data.json")

# Read from file
with open("user_data.json", "r") as f:
    loaded_data = json.load(f)
print("Loaded from file:", loaded_data["name"])

# Parse JSON string
json_text = '{"product": "laptop", "price": 999.99}'
parsed = json.loads(json_text)
print(f"\nProduct: {parsed['product']}, Price: ${parsed['price']}")

# Working with API responses (simulated)
api_response = '''
{
    "users": [
        {"id": 1, "name": "Alice"},
        {"id": 2, "name": "Bob"}
    ],
    "total": 2
}
'''
data = json.loads(api_response)
for user in data["users"]:
    print(f"User {user['id']}: {user['name']}")

📝 JSON ↔ Python

  • dict ↔ object {}
  • list ↔ array []
  • str ↔ string ""
  • int/float ↔ number
  • True/False ↔ true/false
  • None ↔ null

🔧 Key Functions

  • dumps() - Object to string
  • loads() - String to object
  • dump() - Object to file
  • load() - File to object
  • Use indent= for pretty print
API Tip! Most web APIs return JSON. Master this module and you can work with any REST API!

⏱️ time – Time Access and Conversions

👦
Alex
How do I measure how long my code takes to run?
👧
Sam
The time module is your stopwatch! It works with timestamps (seconds since 1970), can pause your program, and measure performance. It's simple but powerful for basic timing needs!
👦
Alex
Why seconds since 1970? That seems random!
👧
Sam
That's the Unix epoch - the birthday of Unix time! January 1, 1970, at 00:00:00 UTC. Let me show you practical uses: 
import time

# Current timestamp
now = time.time()
print(f"Seconds since 1970: {now}")
print(f"That's {now / (365.25 * 24 * 3600):.1f} years!")

# Measure execution time
start = time.time()

# Do some work
total = 0
for i in range(1000000):
    total += i

end = time.time()
print(f"\nCalculation took: {end - start:.4f} seconds")

# More accurate timing with perf_counter
start = time.perf_counter()
time.sleep(0.1)  # Pause for 0.1 seconds
end = time.perf_counter()
print(f"Sleep took: {end - start:.4f} seconds")

# Human-readable time
print(f"\nCurrent time: {time.ctime()}")
print(f"UTC time: {time.gmtime()}")
print(f"Local time: {time.localtime()}")

# Format time nicely
local = time.localtime()
formatted = time.strftime("%Y-%m-%d %H:%M:%S", local)
print(f"Formatted: {formatted}")

# Progress indicator with sleep
print("\nProgress: ", end="")
for i in range(5):
    print(".", end="", flush=True)
    time.sleep(0.2)
print(" Done!")

# CPU time (doesn't include sleep)
cpu_start = time.process_time()
# Do CPU-intensive work
result = sum(range(1000000))
cpu_end = time.process_time()
print(f"\nCPU time used: {cpu_end - cpu_start:.4f} seconds")
Performance Tip! Use time.perf_counter() for measuring short durations - it's more accurate than time.time() for benchmarking!

📅 datetime – Dates and Times

👦
Alex
If we have the time module, why do we need datetime?
👧
Sam
While time deals with raw timestamps, datetime gives you human-friendly dates and times! It's like upgrading from Unix timestamps to a calendar and clock. You can work with dates, do date math, format timestamps beautifully, and handle timezones!
👦
Alex
That sounds much more practical for real applications!
👧
Sam
Absolutely! Let me show you the power of proper date handling: 
from datetime import date, time, datetime, timedelta

# Today's date
today = date.today()
print(f"Today: {today}")
print(f"Year: {today.year}, Month: {today.month}, Day: {today.day}")

# Current date and time
now = datetime.now()
print(f"\nNow: {now}")
print(f"Time: {now.hour}:{now.minute}:{now.second}")

# Create specific dates
birthday = date(2000, 5, 15)
deadline = datetime(2025, 12, 31, 23, 59, 59)
print(f"\nBirthday: {birthday}")
print(f"Deadline: {deadline}")

# Date arithmetic with timedelta
tomorrow = today + timedelta(days=1)
last_week = today - timedelta(weeks=1)
print(f"\nTomorrow: {tomorrow}")
print(f"Last week: {last_week}")

# Calculate age
age = today - birthday
print(f"Age in days: {age.days}")
print(f"Age in years: {age.days // 365}")

# Complex time calculations
meeting = datetime.now() + timedelta(
    days=2, 
    hours=3, 
    minutes=30
)
print(f"\nMeeting in 2 days, 3.5 hours: {meeting}")

# Format dates nicely
print("\nFormatted dates:")
print(now.strftime("%B %d, %Y"))  # January 15, 2025
print(now.strftime("%d/%m/%Y"))   # 15/01/2025
print(now.strftime("%Y-%m-%d %H:%M:%S"))  # ISO format
print(now.strftime("%A, %B %d at %I:%M %p"))  # Tuesday, January 15 at 02:30 PM

# Parse dates from strings
date_string = "2025-03-15 14:30:00"
parsed = datetime.strptime(date_string, "%Y-%m-%d %H:%M:%S")
print(f"\nParsed: {parsed}")

# Day of week
days = ["Monday", "Tuesday", "Wednesday", "Thursday", 
        "Friday", "Saturday", "Sunday"]
print(f"Today is: {days[today.weekday()]}")

# Time until deadline
time_left = deadline - now
print(f"\nTime until deadline: {time_left.days} days")

📆 Core Classes

  • date - Calendar date
  • time - Time of day
  • datetime - Date + time
  • timedelta - Time duration
  • timezone - Timezone info

🎯 Common Uses

  • Log timestamps
  • Schedule tasks
  • Calculate ages/durations
  • Format dates for display
  • Parse date strings
Format Codes! %Y=year, %m=month, %d=day, %H=hour(24), %I=hour(12), %M=minute, %S=second, %A=weekday, %B=month name

🧰 collections – Specialized Data Structures

👦
Alex
Don't we already have lists and dictionaries? Why do we need more?
👧
Sam
The collections module is like having specialized tools instead of just a Swiss Army knife! While lists and dicts are great, sometimes you need a Counter for frequencies, a defaultdict for automatic defaults, or a deque for fast queue operations. These make your code cleaner and faster!
👦
Alex
Show me these specialized tools in action!
👧
Sam
Let's explore the power tools of Python: 
from collections import Counter, defaultdict, namedtuple, deque, OrderedDict

# 1. Counter - Count things automatically!
words = "the quick brown fox jumps over the lazy dog the fox".split()
word_count = Counter(words)
print("Word frequencies:", word_count)
print("Most common 3:", word_count.most_common(3))

# Count letters
letters = Counter("mississippi")
print("\nLetter count:", letters)

# Counter arithmetic
inventory1 = Counter(apples=3, oranges=1)
inventory2 = Counter(apples=1, oranges=2, bananas=2)
total = inventory1 + inventory2
print("Combined inventory:", total)

# 2. defaultdict - Never worry about missing keys!
# Regular dict would raise KeyError
dd = defaultdict(list)
dd["fruits"].append("apple")
dd["fruits"].append("banana")
dd["vegetables"].append("carrot")
print("\ndefaultdict:", dict(dd))

# Group items easily
students = [
    ("Alice", "Math"),
    ("Bob", "Physics"),
    ("Alice", "Physics"),
    ("Charlie", "Math")
]
by_subject = defaultdict(list)
for name, subject in students:
    by_subject[subject].append(name)
print("Students by subject:", dict(by_subject))

# 3. namedtuple - Readable tuples with field names!
# Instead of using indices like point[0], point[1]
Point = namedtuple("Point", ["x", "y"])
p1 = Point(3, 4)
p2 = Point(10, 20)
print(f"\nPoint 1: x={p1.x}, y={p1.y}")
print(f"Distance from origin: {(p1.x**2 + p1.y**2)**0.5}")

# Create a Person record
Person = namedtuple("Person", "name age city")
alice = Person("Alice", 30, "NYC")
print(f"{alice.name} is {alice.age} years old")

# 4. deque - Double-ended queue (fast at both ends!)
# Lists are slow for operations at the beginning
dq = deque([1, 2, 3])
dq.append(4)        # Add to right
dq.appendleft(0)    # Add to left
print("\nDeque:", dq)

# Rotate elements
dq.rotate(2)  # Rotate right by 2
print("Rotated:", dq)

# Limited size deque (keeps last N items)
recent = deque(maxlen=3)
for i in range(5):
    recent.append(i)
    print(f"Added {i}, deque: {list(recent)}")

# 5. OrderedDict - Remember insertion order (useful pre-Python 3.7)
od = OrderedDict()
od["first"] = 1
od["second"] = 2
od["third"] = 3
od.move_to_end("first")  # Move to end
print("\nOrdered after move:", list(od.keys()))

🔧 Specialized Tools

  • Counter - Count frequencies
  • defaultdict - Auto defaults
  • namedtuple - Named fields
  • deque - Fast both ends
  • ChainMap - Multiple dicts

💡 When to Use

  • Counter: Word frequencies, votes
  • defaultdict: Grouping data
  • namedtuple: Readable records
  • deque: Queues, recent history
  • OrderedDict: Preserve order
Performance Tip! deque is O(1) for append/pop at both ends, while list is O(n) at the beginning. Use deque for queues!
